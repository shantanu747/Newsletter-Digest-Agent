# Implementation Plan: Newsletter Digest Agent Enhancements

**Branch**: `001-digest-agent-enhancements` | **Date**: 2026-03-19 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/001-digest-agent-enhancements/spec.md`

## Summary

Enhance the newsletter digest agent with five capabilities: (1) per-sender processing rules with `pass_through` mode for pre-digested newsletters (Superhuman, The Rundown AI, The Code), (2) strict image filtering using HTML heuristics to include only graphs/charts/memes, (3) unread-inbox fetching replacing the 24-hour lookback window, (4) post-delivery mark-as-read + trash of source emails, and (5) continuous 4-hour polling via APScheduler's `IntervalTrigger`. A redesigned Jinja2 email template provides a polished, responsive digest with a table of contents and visual distinction between pass-through and summarized entries.

## Technical Context

**Language/Version**: Python 3.11+
**Primary Dependencies**: `anthropic` SDK, `google-api-python-client` + OAuth 2.0, `beautifulsoup4` + `html2text`, `APScheduler`, `Jinja2`, `PyYAML` + `python-dotenv`, `structlog`, `smtplib` (stdlib)
**Storage**: Stateless files — `token.json` (OAuth), `config/newsletters.yaml` (config). No database.
**Testing**: `pytest` + `pytest-mock`
**Target Platform**: macOS/Linux long-lived process
**Project Type**: Background service / CLI
**Performance Goals**: Process 100 emails in < 20 minutes paced to rate limits; poll every 4 hours
**Constraints**: Anthropic rate limit 0.5 req/sec; Gmail API 1M modify calls/day (non-constraining); OAuth scope upgrade requires one-time user re-authentication
**Scale/Scope**: Single user; ~10–30 newsletters per 4-hour poll after backlog cleared

## Implementation Phases

### Phase A: Core Data Model & Config (Foundation)
Files: `agent/utils/models.py`, `agent/utils/config.py`, `config/newsletters.yaml`

- Add `SenderConfig` dataclass with all per-sender fields
- Extend `Email` with `gmail_message_id` and `is_pass_through`
- Extend `DigestEntry` with `is_pass_through`, `display_name`, `gmail_message_id`
- Add `DigestBatch` dataclass
- Update `AgentConfiguration`: replace `senders: list[str]` with `senders: list[SenderConfig]`; add `poll_interval_hours`, `batch_size`; remove `lookback_hours`
- Update config loader to parse new YAML structure with validation and unknown-key warnings
- Migrate `config/newsletters.yaml` to new format

### Phase B: Gmail Fetcher Upgrades
Files: `agent/fetchers/gmail_fetcher.py`, `scripts/gmail_auth.py`

- Upgrade OAuth scope constant to `gmail.modify` in both files
- Replace time-based query with `from:(...) is:unread in:inbox` query
- Add `mark_as_read(message_id: str)` method
- Add `move_to_trash(message_id: str)` method
- Propagate `gmail_message_id` into `Email` objects during fetch

### Phase C: Image Classification & Pass-Through Parsing
Files: `agent/parsers/email_parser.py`

- Add `_classify_image(img_tag, soup)` method implementing the 6-layer heuristic signal stack (see `data-model.md`)
- Replace existing `_extract_images()` logic with the new classifier
- Add `parse_pass_through(raw_html)` method: lighter boilerplate removal, preserve structural elements (headings, bullets), truncate at 500 words at sentence boundary, append "read original" link

### Phase D: Summarizer Pass-Through Bypass
Files: `agent/summarizer/claude_summarizer.py`

- Accept `SenderConfig` (or `is_pass_through` flag) per email
- For `pass_through` mode: skip Claude API call; return a `Summary` object wrapping the pre-processed pass-through text with `word_count` calculated from the text

### Phase E: Batch Processing & Delivery
Files: `agent/runner.py`, `agent/digest/builder.py`, `agent/digest/delivery.py`

- Update `runner.py`: batch emails (sorted chronologically) into `DigestBatch` objects of size `batch_size`
- For each batch: build digest → send → on success: mark source emails as read → trash source emails
- On delivery failure: skip mark-as-read and trash; log warning; continue to next batch
- Log batch progress: `batch N of M sent, K emails trashed`
- Add `--once` CLI flag to run a single poll and exit

### Phase F: Scheduler Upgrade
Files: `agent/scheduler.py`

- Replace `CronTrigger` with `IntervalTrigger(hours=config.poll_interval_hours)`
- Run job function directly on startup before `scheduler.start()`
- Wrap in `try/except KeyboardInterrupt` for clean shutdown

### Phase G: Template Redesign
Files: `templates/digest.html.j2`

- Table of contents at top (anchor links to each entry)
- Per-entry: sender display name badge, subject as h2, pass-through vs summarized label
- Image grid layout using HTML tables (2-column for 2+ images)
- Styled "Further Reading" link buttons
- Responsive width (max-width 600px, scales on mobile)
- All styles inline for email client compatibility
